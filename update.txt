# Panduan Upgrade Nextcloud Docker dari Versi 31 ke 34

## Persiapan

1. Backup database PostgreSQL.
2. Backup volume Nextcloud (`nextcloud_data`).
3. Pastikan menggunakan image dengan tag versi mayor (misalnya `31-apache`), **bukan** `latest`.

---

# Tahap 1 – Upgrade 31 ke 31 terbaru

Ubah image menjadi:

```yaml
image: nextcloud:31-apache
```

Jalankan:

```bash
docker compose pull
docker compose up -d
docker exec -it nextcloud-app php occ upgrade
docker exec -it nextcloud-app php occ maintenance:repair
```

Pastikan:

```bash
docker exec -it nextcloud-app php occ status
```

Hasilnya:

* version: 31.x terbaru
* maintenance: false
* needsDbUpgrade: false

---

# Tahap 2 – Upgrade ke Nextcloud 32

Ubah image menjadi:

```yaml
image: nextcloud:32-apache
```

Kemudian jalankan:

```bash
docker compose pull
docker compose up -d
docker exec -it nextcloud-app php occ upgrade
docker exec -it nextcloud-app php occ maintenance:repair
```

Pastikan kembali:

```bash
docker exec -it nextcloud-app php occ status
```

# Tahap 4 – Upgrade ke Nextcloud 34

Ubah image menjadi:

```yaml
image: nextcloud:34-apache
```

Kemudian jalankan:

```bash
docker compose pull
docker compose up -d
docker exec -it nextcloud-app php occ upgrade
docker exec -it nextcloud-app php occ maintenance:repair
```

Pastikan status:

```bash
docker exec -it nextcloud-app php occ status
```

Harus menunjukkan:

* maintenance: false
* needsDbUpgrade: false

---

docker exec -it nextcloud-app bash -> php occ maintenance:mode --off

# Setelah setiap upgrade

Jalankan:

```bash
docker exec -it nextcloud-app php occ db:add-missing-indices
docker exec -it nextcloud-app php occ db:add-missing-columns
docker exec -it nextcloud-app php occ db:add-missing-primary-keys
```

Apabila tidak ada yang perlu diperbaiki, Nextcloud akan menampilkan pesan bahwa tidak ada perubahan yang diperlukan.
